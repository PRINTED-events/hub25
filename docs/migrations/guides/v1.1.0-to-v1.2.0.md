# Migration Guide: v1.1.0 -> v1.2.0

## 📋 Overview

Release details are documented in [CHANGELOG.md](../../../CHANGELOG.md).

This guide contains only manual migration steps for user-managed project data after the template update.

Automatic template updates are excluded from this document.

**Estimated Time:** 10-20 minutes

## 🛠️ Manual Changes Required

### 1. Move user-uploaded files to `public/assets/` [Required]

Store user-managed public files in `public/assets/`.

Expected structure:

```text
public/
  assets/
    favicon/
    image-placeholder/
    logo/
    ...
```

### 2. Update file references in content and config [Required]

Change old root public references to `/assets/...` references.

Examples:

- `/logo/light.svg` -> `/assets/logo/light.svg`
- `/favicon/favicon.svg` -> `/assets/favicon/favicon.svg`
- `/image-placeholder/dark.svg` -> `/assets/image-placeholder/dark.svg`

Primary fields to review:

- `general.logo.dark`
- `general.logo.light`
- `general.favicon.dark`
- `general.favicon.light`

Search command:

```bash
grep -RInE "/(logo|favicon|image-placeholder)/" content public
```

## 💡 Optional Recommended Changes

### 1. Migrate icon names to `prefix:name` [Optional | Recommended]

Recommended icon naming format:

- `i-lucide-github` -> `lucide:github`
- `i-mdi-github` -> `mdi:github`
- `i-simple-icons-github` -> `simple-icons:github`

Search command:

```bash
grep -RInE "\bi-(lucide|mdi|simple-icons)-[a-z0-9-]+\b" content public
```

Optional bulk replacement:

```bash
find content public -type f \( -name "*.json" -o -name "*.yml" -o -name "*.yaml" -o -name "*.md" \) -print0 \
  | xargs -0 perl -pi -e 's/\bi-(lucide|mdi|simple-icons)-([a-z0-9][a-z0-9-]*)\b/$1:$2/g'
```

Do not rename:

- CSS classes and ids that use an `i-` prefix.
- Text values that are not icon values.
- Values already using `prefix:name`.

### 2. Keep `@iconify-json/noto` for projects using `noto:*` [Optional | Recommended only when `noto` icons were used in the past]

Install this package only for projects that use `noto:*` icons:

```bash
pnpm add -D @iconify-json/noto
```

Add `noto` to `icon.collections` in `nuxt.config.ts`:

```ts
export default defineNuxtConfig({
  icon: {
    collections: [
      'lucide',
      'mdi',
      'simple-icons',
      'noto',
    ],
  },
})
```

Rename legacy `i-noto-*` values with these commands.<br>
(Step 1 icon name renaming applies to `noto` icons.)

```bash
grep -RInE "\bi-noto-[a-z0-9-]+\b" content public
```

```bash
find content public -type f \( -name "*.json" -o -name "*.yml" -o -name "*.yaml" -o -name "*.md" \) -print0 \
  | xargs -0 perl -pi -e 's/\bi-noto-([a-z0-9][a-z0-9-]*)\b/noto:$1/g'
```

## ✅ Verification

After applying manual changes:

1. Open the site and check logo, favicon, and placeholder images.
1. Open speaker cards and social links and check icon rendering.
1. Run:

```bash
pnpm run test
pnpm run build
```
